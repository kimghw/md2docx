# /recap 명령

인자: $ARGUMENTS

## 목적

이번 세션에서 사용자가 내린 **지시·요구사항만** 추려서 한 파일로 정리한다. 정리 시 **기존 요구사항 문서와의 일관성**과 **세션 내 지시들 간의 논리성**을 함께 점검한다. 실제 정리·점검·저장은 `recap-worker` 서브에이전트가 수행하고, 슬래시 커맨드는 컨텍스트 추출과 디스패치만 담당한다.

## 동작 규칙

1. **인자 처리**:
   - `help` / `-h` / `--help` → 본 명령의 인자(없음 / 파일경로 / `dry`·`preview`)와 동작 설명을 출력하고 종료. 서브에이전트 호출하지 않음.
   - 비어 있음 → 기본 저장 모드 (`user_requirements/recap_<YYYY-MM-DD>.md`).
   - 파일명/경로 → 해당 경로에 저장.
   - `dry` 또는 `preview` → 저장하지 않고 결과만 출력하는 모드.

2. **사전 준비 (오케스트레이터 직접 수행)**:
   - 프로젝트 루트의 `CLAUDE.md`를 Read로 열어 `currentDate`, 저장 위치 규칙, 언어 규칙을 확인한다. `currentDate`가 있으면 그 값을, 없으면 `Bash`로 `date +%Y-%m-%d`를 사용한다.
   - 현재 세션의 user 메시지를 시간순으로 훑어 "지시·요구사항"을 추출한다:
     - 단순 질문·잡담·확인성 응답("응", "ok", "그래")은 제외.
     - 같은 주제로 이어진 지시는 하나로 합치고, 사용자가 도중에 방향을 바꿨다면 최종 결론만 남긴다(방향 전환 사실은 별도로 기록해 서브에이전트에 전달).
   - 인자에 따라 `target_path`(또는 dry 모드 플래그)를 결정한다.

3. **서브에이전트 위임**:
   - `Agent` 도구를 `subagent_type: recap-worker`로 호출한다.
   - `description`은 짧게 "Session recap 작성"으로.
   - `prompt`에 다음 항목을 명시적으로 포함한다(서브에이전트는 세션 컨텍스트를 보지 못한다):
     - `project_root`: 현재 작업 디렉토리 절대경로
     - `current_date`: 위에서 확정한 날짜
     - `mode`: `save` 또는 `dry`
     - `target_path`: save 모드일 때만 (비어 있으면 `user_requirements/recap_<date>.md`, 인자로 주어졌으면 그 경로)
     - `claude_md_excerpt`: CLAUDE.md에서 저장 규칙 관련 부분 발췌(원문 인용)
     - `user_instructions`: 위에서 추출한 사용자 지시 목록(시간순 번호 매김). 각 항목은 "원문 요약 + (필요 시 방향 전환 메모)" 형태.
   - 서브에이전트가 일관성·논리성 점검과 저장을 단독으로 수행한다.

4. **마무리**:
   - 서브에이전트의 `완료_보고` YAML을 받아 사용자에게 다음을 출력한다:
     - 저장된 파일 경로(마크다운 링크)
     - `강조_알림`이 "없음"이 아니면 ⚠ 아이콘과 함께 한 줄로 강조
     - 일관성/논리성 카운트 한 줄 요약(예: "확장 3, 신규 2, 충돌 0 / 미해결 의존 1")
   - dry 모드면 본문도 그대로 사용자에게 전달.
   - 누락 가능성이 있는 항목이 있으면 한 줄로만 알려준다.

## 위임 시 주의

- 서브에이전트는 **현재 세션 대화 내용을 볼 수 없다**. user 메시지 추출은 반드시 오케스트레이터가 prompt에 담아 전달해야 한다.
- 서브에이전트는 기존 `user_requirements/*.md`를 자체적으로 Read하므로 그 내용을 prompt에 다시 담을 필요는 없다.
- 같은 날짜 파일이 존재하면 서브에이전트가 `_2`, `_3` 접미사로 새 파일을 만든다(덮어쓰지 않음). 오케스트레이터는 이를 강제하지 않는다.
