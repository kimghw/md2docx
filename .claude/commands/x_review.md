# Cross Review — 다중 LLM 교차 검토

Claude, Codex, Gemini 를 각각 호출하여 대상을 독립적으로 검토한다. 3개다 호출한다.
각 엔진의 의견을 대조하여 수렴된 사항과 이견이 있는 사항을 분리·정리한다.
검토 범위와 복잡도에 따라 엔진당 호출 수를 조절한다.

## 인자 (Arguments)

- **인자 없음** → 사용자가 지정한 대상(파일·디렉터리·문서 전체)을 검토한다.
- **`change` 또는 `changes`** → **git 변경사항만** 검토한다.
  - 대상 범위: `git status` 로 드러나는 **staged + unstaged + untracked** 변경.
  - 수집 절차:
    1. `git status --porcelain` 으로 변경 파일 목록 확보.
    2. `git diff HEAD` (추적 파일의 staged+unstaged 통합 diff) 와 untracked 파일의 전체 내용을 함께 수집.
    3. 바이너리·대용량 파일은 경로만 나열하고 diff 는 생략.
  - 변경사항이 없으면 "검토할 변경사항 없음" 을 출력하고 엔진을 호출하지 않고 종료.
  - 각 엔진에 전달하는 프롬프트에는 **diff 본문 + 변경 파일 목록** 을 포함하고, "아래 git 변경분만을 대상으로 검토하라" 는 지시를 명시한다.

## 호출 방식 (MCP 우선, CLI 폴백)

각 엔진은 **MCP 서버가 사용 가능하면 MCP로 호출**하고, **없으면 CLI로 폴백**한다.

| 엔진 | MCP (1순위) | CLI 폴백 (2순위) |
|------|-------------|------------------|
| Codex | `mcp__codex__codex` (이어서 답변이 필요하면 `mcp__codex__codex-reply`) | `codex exec "<프롬프트>"` — 비대화형 실행 |
| Gemini | `mcp__gemini__ask-gemini` (긴 응답은 `mcp__gemini__fetch-chunk`) | `gemini -p "<프롬프트>"` — 비대화형 실행 |
| Claude | 현재 세션이 Claude 이므로 자체 검토 수행 | (해당 없음) |

### 처리 절차

1. **가용성 확인**: 사용 가능한 MCP 도구 목록에 해당 서버가 있는지 먼저 점검한다.
   - 있음 → MCP 도구 사용.
   - 없음 → `which codex` / `which gemini` 로 CLI 존재 여부 확인 후 Bash 로 호출.
   - 둘 다 없음 → 해당 엔진은 건너뛰고, 최종 리포트에 "미호출(사유)" 로 명시.
2. **프롬프트 동일성**: MCP 경로든 CLI 경로든 **동일한 프롬프트**를 전달하여 결과가 호출 방식 차이로 편향되지 않도록 한다.
3. **출력 수집**: CLI 폴백의 경우 stdout 을 그대로 기록. 터미널 제어문자가 섞여 있으면 정리 후 비교에 사용.
4. **최종 리포트**에 각 엔진의 호출 방식(MCP/CLI)을 함께 표기하여 재현성을 확보한다.

# 검토 항목

## 일관성 (Consistency)

- 용어·약어가 문서 전체에서 동일하게 사용되는가
- 번호 체계(섹션, 항목, 그림/표 번호)가 연속적이고 참조와 일치하는가
- 동일 개념에 대한 설명이 위치마다 모순되지 않는가

## 무결성 (Completeness)

- 목차·색인에 나열된 항목이 본문에 실제 존재하는가
- 참조(상호 참조, 외부 문서, URL)가 유효하고 대상이 존재하는가
- 빠진 섹션, 미완성 문장, placeholder(TBD/TODO/TBC) 잔존 여부

## 정확성 (Accuracy)

- 수치·단위·수식에 오류가 없는가
- 인용된 표준·규격·법령의 번호와 명칭이 정확한가
- 코드 예시·명령어가 실제 동작과 일치하는가

## 구조·논리 (Structure & Logic)

- 논리 흐름이 자연스럽고 비약이 없는가
- 중복 서술 없이 각 섹션의 역할이 구분되는가
- 전제 → 근거 → 결론 순서가 지켜지는가

## 명확성 (Clarity)

- 모호한 표현(이것, 해당, 적절히 등)이 구체적 대상 없이 사용되지 않는가
- 한 문장이 하나의 의미만 전달하는가
- 대상 독자 수준에 맞는 용어를 사용하는가

## 서식·표기 (Formatting)

- 표·그림의 캡션과 본문 참조가 일치하는가
- 맞춤법·띄어쓰기·문장부호 오류
- 마크다운/LaTeX 등 서식 문법 오류
